# Что реализовано в этом проекте?

1. Шапка сайта (header):
   - При скролле header становится фиксированным.
   - При скролле и при нажатии на пункты меню, пункт меню выделяется активным классом.

2. Реализована функция плавного скролла `smooth scroll`, при добавлении класса `yankor` на ссылку якорь.

3. Реализована функция скрытия текста.

4. Функция скрытия блоков, для планшетов/телефонов. Если в главном блоке, карточек больше 2х, то применятся скрытие, и добавляется кнопка `show more` с лёгким фоновым, затухающим затемнением, под ней.

5. Табы.

6. Фильтрация по категориям. При клике на категорию(на тел/планшетах) `кнопка show more` скроется, и покажется весь контент выбранной вами категории. 

7. Попап.

8. Swiper slider.

9. Бургер-меню.

10. На телефонах/планшетах hover активен сразу, либо будет активен при нажатии на кнопку.

11. Интегрирован Google Maps.

12. Адаптация (Media Queries) для телефонов/планшетов.

13. Masonry сетка.

14. Написан скрипт, который при заходе на сайт проверяет, если браузер поддерживает `.avif`, то все фоны будут `.avif`, если его не поддерживает, то так сделается и с `.webp`, если и его браузер не поддерживает, то будет вставлен исходный фон. Так же при скроле фотографии подгружаются динамически, и в формате который поддерживает браузер, `.avif`, `.webp`, если ничего то вернет исходный формат. А библиотека `lozad` была удалена, потому что не поддерживает тег `picture`, в документации пишет что поддерживает, но погуглив и попрактиковав, оказалось что не поддерживает.

15. Если на элементе, класс active, то к нему hover применятся не будет.

16. Сео оптимизация, и теги для скрин ридеров (accessibility).

17. Для многих кнопок использован формат изображения `svg`.

18. Если нет контента, то `footer` не прилипнет к `header`, а будет прижат к низу.

19. Получение данных с фейкового апи `json placeholder api`, в блоке testimonial. Запросу сделан через библиотеку `axios`. Если слайдер не будет подгружатся(долгая загрузка серва) то вместо контента временно будет спинер.

20. В `папке local`, файлы которые локальные, тоесть они либо общие, либо нацелены на определенный компонент, в `папке response`, получаем данные от сервера. В `папке modules`, скрипты которые можно переиспользовать.

# Технологии, Совместимость, Оптимизация:

1. Flexbox.
2. Grid layout.
3. Применение переменных и миксинов (SCSS).
4. Семантическая верстка / мета-теги (SEO).
5. BEM методология.

<br>

**Также для старых браузеров:**
   - Для CSS добавлены вендорные префиксы.
   - Весь JS код преобразован плагином `Babel`.
   - Преобразование шрифтов в форматы `WOFF` и `WOFF2`.
   - `smooth scroll` добавил через `js` а не через `css`, потому что в старых браузерах, `css` свойство не поддерживается.

# Установка и запуск проекта:

1. Клонировать репозиторий:

```ruby
git clone https://github.com/eugKlim/Bouncy.git
```

2. Установить зависимости:

```ruby
npm install
```

3. Запустить проект:

```ruby
npm run dev
```
